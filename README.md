## Relational vs. Document Database
The first key difference that stood out was the use of documants and collections rather than tables with rows. The documents used by MongoDB were more flexible than the rows, as you could have multiple types within one collection that could include and exclude specific information based on what was needed. Another significant difference is the ability to embed versus reference in the non-relational database, while you can only reference other tables in the relational database.
